About

One engineer, the entire stack.

Full-stack development means more than knowing both ends. It means owning a feature from the database migration through to the pixel — making coherent decisions across data models, API contracts, UI state, and deployment. That's how I've worked for two decades.

At Skywest, where I serve as CTO, I architect and build full-stack systems that handle real production traffic: Django and FastAPI backends, React and Next.js frontends, PostgreSQL and MongoDB data stores, all deployed to AWS using ECS, Lambda, RDS, S3, and CloudFront. I also ship features in Node.js, Laravel, and TypeScript depending on what each project needs.

What full-stack means in practice

  • Design a normalized schema that won't paint you into a corner six months later
  • Build clean REST or GraphQL APIs with thoughtful pagination, error handling, and auth
  • Wire up React components with sensible state management and accessibility baked in
  • Set up infrastructure-as-code, CI/CD, and observability so production doesn't fall over
  • Optimize for performance — query indexing, frontend bundle size, caching strategy
  • Handle auth, billing, file uploads, background jobs, emails — all the unglamorous foundational work

The kind of full-stack work I'm best at

I'm at my strongest building SaaS dashboards, internal tools, data-heavy admin platforms, and AI-powered web apps. If your product has lots of moving data, multiple user roles, and needs to feel fast and polished — that's my sweet spot. I'm also comfortable in fractional CTO and tech-lead roles where I'm responsible for both shipping and setting technical direction.
