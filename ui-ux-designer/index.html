About

Design grounded in how products are built.

I started in design — at Auxilium Studios as a UIUX Designer and 3D Animator, then at Media Direct as a Senior User Experience Designer — before moving fully into engineering leadership. Today, as CTO at Skywest, I still design every interface I ship. The combination is the point: designs that respect technical constraints, and code that respects the user.

I specialize in data-dense product interfaces — dashboards, analytics tools, admin panels, internal tools, and SaaS apps where users come back every day. Those are the interfaces where small UX decisions compound: information density, navigation depth, empty states, loading patterns, keyboard shortcuts. Get them right and the product feels effortless. Get them wrong and users grind through friction every session.

How I approach a design problem

  • Start with the user's actual job. Not personas — the literal task they're trying to complete and the data they need to see.
  • Map the information architecture first. Routes, navigation, what lives where. Pixels last.
  • Design in components. A coherent design system beats one beautiful screen.
  • Prototype with real data. Lorem ipsum hides the hardest problems.
  • Build for the edges. Empty states, error states, long content, loading — they're not afterthoughts.
  • Accessibility from day one. Keyboard navigation, contrast, focus states, screen-reader support.

What I design

SaaS dashboards, admin platforms, AI tools, onboarding flows, marketing sites, and complete design systems. I work in Figma, prototype in code when needed, and hand off (or implement) production-ready React and CSS. I've shipped designs for reporting platforms, 3D marketplaces, financial networks, and AI agents — across both light and dark themes, mobile and desktop.
